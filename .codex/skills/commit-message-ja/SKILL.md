# Commit Message JA

## 概要

PressWatch のコミットメッセージを作成・提案するときのプロジェクト固有ルール。コミットはレビューしやすい単位に分け、メッセージは日本語で簡潔に書く。

## 方針

- コミットメッセージは日本語で記述する
- 原則として Conventional Commits 形式を使う
- 変更内容が明確に伝わる場合は1行の subject のみにする
- 背景、制約、判断理由が1行では伝わりにくい場合のみ本文を追加する
- subject は命令形にこだわらず、日本語として自然な表現を優先する
- subject の末尾に句点を付けない
- 無関係な変更を1つのコミットに混ぜない
- ステージ済みの内容を確認してから、実際の差分に合うメッセージを作る

## 形式

```text
<type>(<scope>): <日本語の要約>
```

`scope` は分かる場合だけ付ける。迷う場合は無理に付けない。

## type の目安

- `feat`: ユーザーや利用側から見える機能追加
- `fix`: バグ修正
- `docs`: ドキュメントや作法の追加・修正
- `test`: テストの追加・修正
- `refactor`: 振る舞いを変えない整理
- `chore`: 開発補助、設定、メタデータなど

## 例

```text
feat(scraper): 環境省の報道発表取得を追加
```

```text
docs: コミットメッセージ作法を追加
```

```text
test(scraper): 異常日付のパーステストを追加
```

## 本文を付ける例

```text
feat(scraper): 環境省の報道発表取得を追加

報道発表一覧HTMLから発表項目と月別アーカイブリンクを抽出する処理を追加した。
CLI からURL取得または保存済みHTMLの解析を実行できるようにし、日付形式の異常系をテストで固定した。
```

## 確認観点

- ステージ済みのファイルだけを説明しているか
- type と scope が差分に合っているか
- 1行で十分なコミットに本文を付けすぎていないか
- 複数コミットに分けるべき変更が混ざっていないか
