# PressWatch - Tasks

## 方針

- 1タスク = 1責務 を意識する
- 各タスクはレビューしやすい粒度に分ける
- まずはMVP完成を優先する
- いきなり全部作らず、土台 → データ取得 → 保存 → API → 画面 の順で進める
- Codex には小さな単位で依頼し、差分ベースでレビューする

## Codexへ依頼する時の基本ルール

- 一度に大きく作らせすぎない
- 1回の依頼で1責務に絞る
- 差分ベースでレビューする
- 要件にない機能を勝手に広げない
- 不要なリファクタを避ける
- 何を作るか、何を作らないかを毎回明示する
- main ブランチとの差分で確認しやすい単位を保つ
